Snort3 IDS/IPS 설치 가이드

🛡️ Snort3 Linux 기반에서 구현하기

이번 포스트에서는 Snort3 IDS/IPS를 Ubuntu/Debian 환경에서 구축하는 방법에 대해 자세히 알아보겠습니다.

📋 Snort3란?

Snort3는 오픈소스 기반의 네트워크 침입 탐지 및 방지 시스템(IDS/IPS)으로, 실시간 트래픽 분석과 패킷 로깅 기능을 제공합니다.

💡 주요 특징:
• 실시간 네트워크 트래픽 분석
• 유연한 룰 기반 탐지 엔진
• 다양한 출력 플러그인 지원
• 고성능 패킷 처리

🛠️ 사전 요구사항 설치

Snort3 컴파일에 필요한 의존성 패키지들을 설치합니다:

apt install build-essential libpcap-dev libpcre3-dev libnet1-dev zlib1g-dev luajit hwloc libdumbnet-dev bison flex liblzma-dev openssl libssl-dev pkg-config libhwloc-dev cmake cpputest libsqlite3-dev uuid-dev libcmocka-dev libnetfilter-queue-dev libmnl-dev autotools-dev libluajit-5.1-dev libunwind-dev libfl-dev -y

📥 libdaq 설치

Data AcQuisition 라이브러리를 설치합니다:

Git 설치 및 소스 다운로드

apt install -y git
git clone https://github.com/snort3/libdaq.git

libdaq 컴파일 및 설치

cd libdaq
./bootstrap
./configure
make && make install

⚡ gperftools 설치

성능 및 메모리 관리 유틸리티를 설치합니다:

wget https://github.com/gperftools/gperftools/releases/download/gperftools-2.13/gperftools-2.13.tar.gz
tar xzf gperftools-2.13.tar.gz
cd gperftools-2.13
./configure 
make && make install
📝 참고:
• make: 소스코드를 컴파일
• make install: 컴파일된 파일을 시스템에 설치

🔧 Snort3 설치

소스코드 다운로드

cd ~
wget https://github.com/snort3/snort3/archive/refs/heads/master.zip
apt install unzip
unzip master.zip
cd snort3-master/

Snort3 컴파일

./configure_cmake.sh --prefix=/usr/local --enable-tcmalloc
cd build
make && make install

라이브러리 링크 업데이트

ldconfig

설치 확인

snort -V
설치가 성공하면 Snort++ 버전 정보가 출력됩니다.

⚙️ 네트워크 인터페이스 설정

무작위 모드(Promiscuous Mode) 설정

ip link set dev enp0s3 promisc on

설정 확인

ip addr  # PROMISC 플래그 확인

ethtool 설치 및 설정

apt install -y ethtool
ethtool -K enp0s3 gro off lro off
⚠️ 주의: enp0s3를 실제 네트워크 인터페이스명으로 변경하세요.

🚀 시스템 서비스 설정

서비스 파일 생성

touch /etc/systemd/system/snort3-nic.service
vi /etc/systemd/system/snort3-nic.service

서비스 파일 내용

[Unit]
Description=Set Snort 3 NIC in promiscuous mode and Disable GRO, LRO on boot
After=network.target
 
[Service]
Type=oneshot
ExecStart=/usr/sbin/ip link set dev enp0s3 promisc on
ExecStart=/usr/sbin/ethtool -K enp0s3 gro off lro off
TimeoutStartSec=0
RemainAfterExit=yes
 
[Install]
WantedBy=default.target

서비스 등록 및 실행

systemctl daemon-reload
systemctl start snort3-nic
systemctl enable snort3-nic
systemctl status snort3-nic

📜 룰 파일 설정

룰 디렉터리 생성

mkdir /usr/local/etc/snort/rules

커뮤니티 룰 다운로드

wget -qO- https://www.snort.org/downloads/community/snort3-community-rules.tar.gz | tar xz -C /usr/local/etc/snort/rules/

설정 파일 편집

vi /usr/local/etc/snort/snort.lua

주요 설정 변경 사항:

  • 24번 라인: HOME_NET = '192.168.0.0/16'
  • 28번 라인: EXTERNAL_NET = '!$HOME_NET'
  • 183번 라인 ips 섹션에 룰 파일 경로 추가
ips =
{
    variables = default_variables,
    rules = [[
      include /usr/local/etc/snort/rules/local.rules
      include /usr/local/etc/snort/rules/snort3-community-rules/snort3-community.rules
    ]]
}

✏️ 사용자 정의 룰 작성

로컬 룰 파일 생성

touch /usr/local/etc/snort/rules/local.rules
vi /usr/local/etc/snort/rules/local.rules

예제 룰

# ICMP 탐지 룰
alert icmp any any -> $HOME_NET any (msg:"Incoming ICMP"; sid:1000001; rev:1;)

# 8.8.8.8 Ping 탐지
alert icmp any any -> any any (msg:"8.8.8.8 Ping"; sid:1000002; rev:1;)

# HTTPS 접속 탐지
alert tcp any any -> any 443 (msg:"Web Attacking"; sid:1000003; rev:1;)

# 특정 IP에서 HTTP 접속 탐지
alert tcp 192.168.1.17 any -> any 80 (msg:"Web Test"; sid:1000004; rev:1;)
📝 룰 구조 설명:
• alert: 액션 (경고 및 탐지)
• tcp/icmp: 프로토콜
• 소스IP 소스포트 -> 목적지IP 목적지포트
• msg: 로그 메시지
• sid: 시그니처 ID
• rev: 룰 버전

🔍 OpenAppId 설치 (선택사항)

애플리케이션 탐지 기능을 위한 OpenAppId를 설치합니다:

wget https://www.snort.org/downloads/openappid/33380 -O OpenAppId-33380.tgz
tar -xzvf OpenAppId-33380.tgz
cp -R odp /usr/local/lib/

snort.lua 파일에 AppId 설정 추가

appid =
{
    app_detector_dir = '/usr/local/lib',
    log_stats = true,
}

appid_listener =
{
    json_logging = true,
    file = "/var/log/snort/appid-output.log",
}

▶️ Snort3 실행

설정 파일 검증

snort -c /usr/local/etc/snort/snort.lua

탐지 모드로 실행

snort -c /usr/local/etc/snort/snort.lua -R /usr/local/etc/snort/rules/local.rules -i enp0s3 -A alert_fast -s 65535 -k none
🎯 실행 옵션 설명:
• -c: 설정 파일 경로
• -R: 룰 파일 경로
• -i: 모니터링할 인터페이스
• -A: 알림 모드
• -s: 패킷 크기
• -k: 체크섬 검사 옵션

이제 Snort3가 실시간으로 네트워크 트래픽을 모니터링하고, 정의된 룰에 따라 침입을 탐지합니다.

Suricata IDS/IPS 설치 가이드

🔐 Suricata Rocky Linux 9 기반에서 구현하기

이번 포스트에서는 Suricata IDS/IPS를 Rocky Linux 9 환경에서 구축하는 방법에 대해 자세히 알아보겠습니다.

📋 Suricata란?

Suricata는 고성능 네트워크 IDS, IPS 및 네트워크 보안 모니터링 엔진으로, 멀티스레딩 아키텍처를 통해 높은 처리량을 제공합니다.

💡 주요 특징:
• 멀티스레드 처리로 고성능
• JSON 형태의 상세한 로깅
• Lua 스크립팅 지원
• 파일 추출 및 TLS 인증서 로깅

🛠️ 시스템 초기 설정

시스템 업데이트

dnf update -y
dnf upgrade -y

EPEL 저장소 설치

dnf install -y epel-release

Suricata 설치

dnf install -y suricata

🌐 네트워크 인터페이스 확인

시스템의 네트워크 인터페이스를 확인합니다:

ip --brief addr
이 명령어로 확인한 인터페이스명을 다음 설정에서 사용합니다.

⚙️ Suricata 환경설정

메인 설정 파일 편집

vi /etc/suricata/suricata.yaml

주요 설정 변경사항:

  • HOME_NET: 내부 네트워크 대역 설정 (예: 192.168.0.0/16)
  • 622번 라인: interface를 실제 인터페이스명으로 변경

서비스 옵션 설정

vi /etc/sysconfig/suricata

OPTIONS 항목에서 인터페이스를 실제 인터페이스명으로 변경합니다.

⚠️ 주의: 네트워크 인터페이스명은 시스템마다 다를 수 있습니다. 정확한 인터페이스명을 사용하세요.

📜 룰 업데이트 및 관리

기본 룰 업데이트

suricata-update

사용 가능한 룰 소스 확인

suricata-update list-sources

추가 룰셋 활성화

suricata-update enable-source et/open
suricata-update

기본 룰 파일 확인

ls -al /var/lib/suricata/rules/
cat /var/lib/suricata/rules/suricata.rules

🚀 서비스 관리

설정 파일 검증

suricata -c /etc/suricata/suricata.yaml

서비스 상태 확인

systemctl status suricata

서비스 시작 및 자동 시작 설정

systemctl enable --now suricata

서비스 재시작

systemctl restart suricata

✏️ 사용자 정의 룰 생성

로컬 룰 파일 생성

vi /etc/suricata/rules/local.rules

메인 설정 파일에 로컬 룰 추가

vi /etc/suricata/suricata.yaml

2187~2189번 라인 근처에 다음 내용을 추가:

rule-files:
  - /etc/suricata/rules/local.rules
  - suricata.rules

설정 검증

suricata -T -c /etc/suricata/suricata.yaml -v
📝 룰 작성 팁:
• 모든 사용자 정의 룰은 /etc/suricata/rules/ 경로에 위치
• 룰 추가 후 반드시 설정 검증 실행
• 룰 변경 후 서비스 재시작 필요
