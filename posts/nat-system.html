🖥️ GNS3 Topology NAT(Network Address Translation) 구현하기

이번 포스트에서는 NAT(Network Address Translation)를 구축하는 방법에 대해 자세히 알아보겠습니다.

📋 NAT란?

NAT는 네트워크에서 IP 패킷의 주소 정보를 변경하여 네트워크 트래픽을 주고받는 기술입니다.
주로 사설 IP 주소를 공인 IP 주소로 변환하여 외부와 통신하게 도와줍니다.

🚀 주요 특징

  • IP 주소 절약: IPv4 주소 부족 문제를 해결하는 데 도움을 줍니다.
  • 보안 강화: 내부 네트워크 주소 노출을 막아 보안을 강화합니다.
  • 유연한 네트워크 구성: 동적 IP 관리 및 트래픽 제어가 용이합니다.

🛠️ NAT 설정 예시 (Cisco)

! NAT 1:1 - Static NAT 예시
ip nat inside source static 192.168.100.100 1.1.1.1

! 인터페이스 설정
interface FastEthernet0/0
 ip nat inside
interface FastEthernet0/1
 ip nat outside

! NAT 통계 확인
show ip nat statistics

! PAT (Port Address Translation - Overloading) 설정
access-list 10 permit 192.168.100.0 0.0.0.255
ip nat pool admin 1.1.1.1 1.1.1.254 netmask 255.255.255.0
ip nat inside source list 10 pool admin overload

위 설정은 내부 네트워크(192.168.100.0/24)를 공인 IP 대역(1.1.1.1 ~ 1.1.1.254)으로 변환하며,
PAT(overloading)을 통해 여러 내부 IP가 하나 또는 여러 공인 IP를 공유하여 인터넷에 접속할 수 있습니다.
